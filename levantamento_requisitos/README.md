# 📋 Levantamento de Requisitos: Engenharia de Software

Este guia abrange as fases de elicitação, análise e documentação necessárias para o desenvolvimento de sistemas eficazes.

---

## 📚 1. Classificação de Requisitos
*Entendendo o que o software deve ser e como ele deve se comportar.*

- **Requisitos Funcionais (RF):**
  - Definem as funções específicas do sistema.
  - *Exemplo:* "O sistema deve permitir o cadastro de novos usuários."
- **Requisitos Não Funcionais (RNF):**
  - Definem critérios de operação e qualidades do sistema (desempenho, segurança, usabilidade).
  - *Exemplo:* "O sistema deve criptografar todas as senhas (Segurança)" ou "O sistema deve responder em até 1s (Performance)".

---

## 🔍 2. Técnicas de Elicitação (Coleta)
*Métodos práticos para descobrir as necessidades do cliente.*

- **Brainstorm (Tempestade de Ideias):**
  - Sessões criativas em grupo para explorar funcionalidades inovadoras sem julgamentos prévios.
- **Entrevistas:**
  - Conversas diretas com stakeholders. Podem ser estruturadas (perguntas fixas) ou não estruturadas (abertas).
- **Prototipagem:**
  - Criação de modelos visuais (baixa ou alta fidelidade) para validar requisitos com o usuário antes da codificação.

---

## 📊 3. Modelagem e Diagramas
*Transformando palavras em representações visuais.*

- **Diagrama de Casos de Uso (UML):** Representa a interação dos atores (usuários/sistemas) com as funcionalidades.
- **Diagrama de Fluxo de Dados (DFD):** Mostra como a informação transita dentro do sistema.
- **Protótipos de Tela:** Desenhos que auxiliam na compreensão do fluxo de navegação.

---

## 📄 4. Relatórios Técnicos e Documentação
*A formalização do que foi levantado.*

- **Documento de Visão:** Define o escopo geral e os objetivos do projeto.
- **Especificação de Requisitos de Software (ERS):** O contrato técnico contendo todos os RFs e RNFs detalhados.
- **Relatório de Elicitação:** Registro das conclusões obtidas após entrevistas e sessões de brainstorm.

---

## 🚀 5. Exercício Prático
**Estudo de Caso: App de Delivery**
1. Realize um **Brainstorm** para listar 5 funções inovadoras.
2. Classifique 3 **Requisitos Funcionais** e 2 **Não Funcionais**.
3. Desenhe um **Protótipo** de baixa fidelidade (papel/wireframe) da tela de checkout.
4. Redija um pequeno **Relatório Técnico** justificando a escolha das tecnologias baseada nos requisitos não funcionais.
